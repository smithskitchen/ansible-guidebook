Ansible Learning Roadmap: Beginner to Expert

Beginner Level
1. Introduction & Setup
Key Concepts:
  • Overview of Ansible: Definition, use cases, and architecture
  • Understanding control node, managed nodes, and inventory
  • YAML basics, the syntax used for playbooks
Milestones:
  • Install Ansible (on your system or a virtual machine)
  • Connect to remote hosts using SSH
Practical Exercises:
  • Install Ansible on your workstation
  • Create an inventory file with localhost and/or another VM
  • Run a ping module: ansible all -m ping to verify host connections
2. Basic Ad-Hoc Commands
Key Concepts:
  • Using Ansible modules (ping, command, shell, copy, file, etc.)
  • Running simple ad-hoc tasks on inventory hosts
Milestones:
  • Use ad-hoc commands to perform operations (creating files, installing packages)
Practical Exercises:
  • Create a directory on all managed hosts
  • Install a package (e.g., httpd or nginx) across all hosts
3. Playbooks & Variables
Key Concepts:
  • Structure of playbooks (hosts, tasks, handlers)
  • Use of variables, facts, and simple templating
Milestones:
  • Write your first multiple-task playbook
  • Use variables in playbooks
Practical Exercises:
  • Write a playbook to install Apache/Nginx and ensure its service is running
  • Use variables for the service name

Intermediate Level
4. Roles & Reusability
Key Concepts:
  • Organizing playbooks using roles
  • Understanding role directory structure, variable precedence, and scope
Milestones:
  • Create and use roles for modular configuration
Practical Exercises:
  • Refactor a previous web server playbook into a role, and use it for multiple server types
5. Templates & Jinja2
Key Concepts:
  • Using Jinja2 templates to create configuration files
  • Conditional logic and filters in templates
Milestones:
  • Deploy a templated configuration file customized per host or group
Practical Exercises:
  • Create a Jinja2 template for Nginx virtual hosts; deploy to multiple hosts with varying content
6. Inventories: Static & Dynamic
Key Concepts:
  • INI and YAML inventory formats
  • Dynamic inventory scripts/plugins
Milestones:
  • Configure and use a dynamic inventory (e.g., AWS, Azure, or custom script)
Practical Exercises:
  • Use the AWS EC2 plugin to discover and manage cloud hosts
7. Error Handling & Idempotence
Key Concepts:
  • Error handling (ignore_errors, failed_when, registered variables)
  • Idempotence: Playbooks producing the same results on repeated runs
Milestones:
  • Handle task failures and use handlers to trigger notifications
Practical Exercises:
  • Write a playbook to restart services only if a config change occurs and handle failures in service tasks
Advanced Level
8. Ansible Galaxy & Collections
Key Concepts:
  • Using community-maintained roles and collections
  • Publishing your own roles
Milestones:
  • Use a Galaxy role in a project
  • Publish a simple role to Galaxy
Practical Exercises:
  • Use a public role (e.g., geerlingguy.mysql) to install MySQL on test hosts
9. Advanced Variables & Fact Gathering
Key Concepts:
  • Host, group, and global variables
  • Ansible Vault for secret management
  • Advanced facts (custom facts, setup module)
Milestones:
  • Manage secrets securely with Ansible Vault
  • Write custom fact scripts
Practical Exercises:
  • Encrypt database passwords with Ansible Vault
  • Use custom facts for host-specific configuration
10. Strategies, Lookups, and Plugins
Key Concepts:
  • Execution strategies (linear, free)
  • Lookup plugins, filters, and callback plugins
  • Writing custom plugins
Milestones:
  • Use lookups to query external data (files, services)
  • Configure callback plugins for event-driven behavior
Practical Exercises:
  • Use a lookup to add users from a CSV on target servers
11. Testing & CI/CD Integration
Key Concepts:
  • Testing roles and playbooks with Molecule
  • Integrating Ansible with continuous integration tools (Jenkins, GitHub Actions)
Milestones:
  • Write Molecule tests for roles
  • Trigger playbook runs from a Git commit
Practical Exercises:
  • Use GitHub Actions to lint/test your playbooks on push
12. Custom Modules & Contributing
Key Concepts:
  • Writing Ansible modules (in Python)
  • Contributing to Ansible open-source
Milestones:
  • Create and use a custom module
  • Submit an issue or pull request to an Ansible collection
Practical Exercises:
  • Write a custom Python module for a task not covered by core modules
Expert Level
13. Scalable Deployments & AWX/Automation Controller
Key Concepts:
  • Using Ansible at scale
  • Using AWX or Ansible Automation Platform
Milestones:
  • Set up AWX locally or in the cloud
  • Run jobs and manage inventories from AWX
Practical Exercises:
  • Create a workflow job with AWX that includes scheduling, notifications, and access control
14. Enterprise Patterns & Best Practices
Key Concepts:
  • Organizing large-scale playbooks and roles
  • Access control, RBAC, and enterprise secrets management
  • Audit and compliance
Milestones:
  • Design a project structure suitable for a large team/environment
  • Implement GitOps with Ansible
Practical Exercises:
  • Architect and document an Ansible setup supporting hundreds of hosts and RBAC
